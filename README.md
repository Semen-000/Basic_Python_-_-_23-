# Функция print(): Шпаргалка
## Аргументы
Объекты, которые передаём в функцию (строки, числа, списки)

Разделяются запятой

Ошибка при отсутствии запятой

## Параметры (имеют значения по умолчанию)
## Параметр end
Значение по умолчанию: "\n" (перевод строки)

Добавляется после последнего аргумента

Пример: print("Привет", end=" ") print("Мир") → "Привет Мир"

## Параметр sep
Значение по умолчанию: " " (пробел)

Разделяет аргументы между собой

Пример: print("Я", "ты", "он", sep="-") → "Я-ты-он"

## Совместное использование sep и end
print("A", "B", "C", sep="...", end="!!!") → "A...B...C!!!"

## Использование переменных
razdelitel = " | "
print("Раз", "Два", "Три", sep=razdelitel) → "Раз | Два | Три"

## Важные моменты
sep и end указываются в конце вызова функции

Запись без пробелов: sep='_' (соответствует PEP 8)

Принимают только строковые значения

Не путать /n (слеш) и \n (обратный слеш)

## Сводка
| Параметр | Область действия | Значение по умолчанию |
|----------|-----------------|----------------------|
| sep | Между аргументами | пробел " " |
| end | После последнего аргумента | перевод строки "\n" |

# Работа с длинными строками в Python
## Что такое длинная строка кода?
PEP 8 рекомендует ограничивать длину строки кода до 79 символов. Если строка превышает это значение, её можно считать длинной.

Пример длинной строки кода (96 символов):
```bash
long_string = "В Python рекомендуется использовать строки, длина которых не более 79 символов."
```
**Важно различать:**

- **Строка кода** — весь код на одной линии в редакторе

- **Строка** — тип данных (набор символов в кавычках)

## Виды длинных строк
**Текст из одного абзаца:**
```bash
# Вся строка в одной строке кода
text = "В Python рекомендуется использовать строки, длина которых не более 79 символов. Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода."
```
**Текст из нескольких абзацев:**
```bash
text = "В Python рекомендуется использовать строки, длина которых не более 79 символов.\nЭто ограничение обусловлено необходимостью обеспечения удобочитаемости кода.\nЭто улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов."
```
## Способы переноса строк
## 1. Вертикальное выравнивание
Текст помещается в круглые скобки, части переносятся и выравниваются по началу:
```bash
long_str = (
    "В Python рекомендуется использовать строки, длина которых не более 79 символов. "
    "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода. "
    "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов."
)

print(long_str)
# В Python рекомендуется использовать строки, длина которых не более 79 символов. Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода. Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.
```
**Важные моменты:**

- Строки в скобках автоматически соединяются в одну

- Пробелы в конце строк нужны для правильного соединения

- Запятые не ставятся (иначе получится кортеж)

## 2. Висячий отступ
Все элементы переносятся на новую строку с отступом в 4 пробела:
```bash
long_str = (
    "В Python рекомендуется использовать строки, длина которых не более 79 символов. "
    "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода. "
    "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов."
)
```
## Сразу в печать
**Вариант с соединением строк:**
```bash
print("В Python рекомендуется использовать строки, длина которых не более 79 символов. "
      "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода. "
      "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.")
```
**Вариант с использованием sep (рекомендуется):**
```bash
print("В Python рекомендуется использовать строки, длина которых не более 79 символов.",
      "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода.",
      "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.")
# В Python рекомендуется использовать строки, длина которых не более 79 символов. Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода. Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.
```
## Работа с текстом из нескольких абзацев
## Способ 1. Тройные кавычки
```bash
long_string = """В Python рекомендуется использовать строки, длина которых не более 79 символов.
Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода.
Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов."""

print(long_string)
# В Python рекомендуется использовать строки, длина которых не более 79 символов.
# Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода.
# Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.
```
## Способ 2. Висячий отступ с \n
```bash
long_string = (
    "В Python рекомендуется использовать строки, длина которых не более 79 символов.\n"
    "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода.\n"
    "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов."
)
print(long_string)
```
## Способ 3. Print с sep='\n' (рекомендуется)
```bash
print(
    "В Python рекомендуется использовать строки, длина которых не более 79 символов.",
    "Это ограничение обусловлено необходимостью обеспечения удобочитаемости кода.",
    "Это улучшает чтение кода в редакторах с шириной окна, установленной на 80 символов.",
    sep='\n'
)
```
## Обратная косая черта \ (не рекомендуется)
PEP 8 не рекомендует этот способ, но он возможен:
```bash
long_string = "Добро пожаловать на курс: Осознанный Python. "\
              "Данный курс подарит вам много интересного "\
              "в сфере программирования."

print(long_string)
# Добро пожаловать на курс: Осознанный Python. Данный курс подарит вам много интересного в сфере программирования.
```
**Ограничения:**

- После \ нельзя ставить пробелы или комментарии

- Лучше использовать скобки вместо \

## Сравнение способов
| Способ | Когда использовать | Преимущества | Недостатки |
|--------|-------------------|--------------|------------|
| Вертикальное выравнивание | Короткие строки, одна переменная | Простота | Зависит от длины имени |
| Висячий отступ | Универсально | Гибкость, читаемость | Требует больше строк |
| Тройные кавычки | Много абзацев | Сохраняет формат | Длина строк может превышать 79 |
| sep='\n' в print | Вывод на экран | Явное указание разделителя | Только для вывода |
| Обратная косая черта \ | Теоретически возможно | - | Не рекомендуется PEP 8 |
## Рекомендации PEP 8
- Для тройных кавычек использовать двойные кавычки """

- Предпочитать круглые скобки обратному слэшу \

- Допустимо превышение 79 символов до 99 для улучшения читаемости

## Итог
При работе с длинными строками:

**1.** Используйте круглые скобки для переноса

**2.** Для вывода на экран применяйте sep='\n'

**3.** Для многоабзацевого текста подходят тройные кавычки

# Специальные символы в Python
## Основные специальные символы
Специальные символы начинаются с обратной косой черты \ и изменяют поведение строки.

| Символ | Название | Действие |
|--------|----------|----------|
| ```\n``` | Перевод строки | Переносит курсор на новую строку |
| ```\t``` | Табуляция | Создаёт отступ (обычно 4 пробела) |
| ```\r``` | Возврат каретки | Возвращает курсор в начало строки |

## Примеры использования
```bash
print("Привет!\nКак дела?")
# Результат:
# Привет!
# Как дела?

print("Привет!\tКак дела?")
# Результат:
# Привет!    Как дела?
```
## Проблемы со специальными символами
Когда обратная косая черта ```\``` стоит перед буквами n, t, r, Python воспринимает их как специальные символы, а не как обычный текст.

**Проблемный пример:**
```bash
print("C:\run\new\transport")
# Результат (не то, что ожидалось):
# C:    un
# ew    ransport
```
## Решения проблем
## Решение 1. Экранирование обратным слешем
Используйте двойной обратный слеш ```\\```, чтобы отменить действие специального символа.
```bash
print("C:\\main\\new\\transport")
# Результат:
# C:\main\new\transport
```
## Решение 2. Сырые строки (префикс r)
Префикс ```r``` (raw) перед строкой отключает действие всех специальных символов.
```bash
print(r"C:\run\new\transport")
# Результат:
# C:\run\new\transport
```
**Префикс r работает с любыми кавычками:**
```bash
print(r'C:\run\new\transport')
print(r"C:\run\new\transport")
print(r"""C:\run\new\transport""")
print(r'''C:\run\new\transport''')
```
**Присвоение сырой строки переменной:**
```bash
string1 = r"C:\run\new\transport"
```
## Важные ограничения сырых строк
## 1. Не решают проблему кавычек
```bash
s = r"Это "строка" с кавычками"  # Ошибка!
```
## 2. Не могут заканчиваться на нечётное количество обратных слешей
```bash
s = r"\"  # Ошибка: SyntaxError
```
Python не может определить, является ли последний слеш частью строки или экранирующим символом.

| Способ | Синтаксис | Когда использовать | Преимущества |
|--------|-----------|-------------------|--------------|
| Экранирование | ````"C:\\main\\new"```` | Только несколько символов | Точный контроль |
| Сырые строки | ````r"C:\main\new"```` | Пути и регулярные выражения | Лаконичность, читаемость |

## Рекомендации
- Для путей к файлам используйте сырые строки с префиксом ```r```

- При работе с регулярными выражениями также предпочтительны сырые строки

- Если нужно экранировать отдельные символы, используйте двойной слеш ```\\```

- Помните, что сырые строки не решают проблему с кавычками внутри строки

Типичные ошибки начинающих

❌ Неправильно:
```bash
print("C:\Users\Name\Documents")
```
✅ Правильно:
```bash
print(r"C:\Users\Name\Documents")
# или
print("C:\\Users\\Name\\Documents")
```
## Итог
- Специальные символы (```\n```, ```\t```, ```\r```) изменяют поведение строк

- Обратная косая черта может создавать неожиданные проблемы

- Для решения используйте **экранирование** (```\\```) или **сырые строки** (префикс ```r```)

- Сырые строки — более современный и лаконичный подход

- Внимательно читайте сообщения об ошибках — они помогут найти проблему

  # Функции ord() и chr()
## Основные понятия
**Unicode** — стандарт кодирования, где каждый символ имеет уникальный числовой код.

| Функция | Назначение | Пример |
|---------|------------|--------|
| ```ord(символ)``` | Возвращает код символа | ```ord('A')``` → 65` |
| ```chr(код)``` | Возвращает символ по коду | ```chr(65)``` → 'A'` |

## Таблица кодов английских букв
| Регистр | Диапазон букв | Диапазон кодов |
|---------|---------------|----------------|
| Заглавные (A-Z) | A (1-я) ... Z (26-я) | 65 ... 90 |
| Строчные (a-z) | a (1-я) ... z (26-я) | 97 ... 122 |
## Таблица кодов русских букв
| Регистр | Диапазон букв | Диапазон кодов |
|---------|---------------|----------------|
| Заглавные (А-Я) | А (1-я) ... Я (33-я) | 1040 ... 1071 |
| Строчные (а-я) | а (1-я) ... я (33-я) | 1072 ... 1103 |
| Отдельно | Ё | 1025 |
| | ё | 1105 |

## Примеры использования
```bash
# Проверка регистра
print(ord('П') > ord('п'))  # False (1055 > 1087? Нет)

# Преобразование регистра (латиница)
print(chr(ord('a') - 32))  # 'A' (97 - 32 = 65)

# Сдвиг символа (шифр)
print(chr(ord('A') + 3))   # 'D'
```
## Важно
- Разный регистр = разные коды

- Коды идут по алфавиту (A=65, B=66, C=67...)

- Русский алфавит имеет разрыв после буквы "Е" (Ё имеет отдельный код)

# *args и **kwargs
## 1. Запаковка аргументов (в определении функции)

| Синтаксис | Назначение | Результат |
|-----------|------------|-----------|
| `*args` | Произвольное количество **позиционных** аргументов | Кортеж `args` |
| `**kwargs` | Произвольное количество **именованных** аргументов | Словарь `kwargs` |

Примеры:
```bash
def func(*args):
    print(args)  # (1, 2, 3)

def func(**kwargs):
    print(kwargs)  # {'a': 1, 'b': 2}
```
## 2. Правила расположения параметров
Правильный порядок:
```bash
позиционные → *args → именованные (со значениями по умолчанию) → **kwargs
```
Пример:
```bash
def func(a, b, *args, c=10, d=20, **kwargs):
    pass
```
## 3. Распаковка аргументов (при вызове функции)

| Синтаксис | Действие |
|-----------|----------|
| `*iterable` | Распаковывает итерируемый объект в позиционные аргументы |
| `**dict` | Распаковывает словарь в именованные аргументы |

Примеры:
```bash
print(*[1, 2, 3])           # 1 2 3 (как print(1, 2, 3))
func(**{'a': 1, 'b': 2})    # func(a=1, b=2)
```
## 4. Важные моменты
- **Имена не важны** — магия в звёздочках (```*котик```, ```**пёсик``` тоже работают)

- **args** — сокращение от "arguments" (позиционные)

- **kwargs** — сокращение от "keyword arguments" (именованные)

При распаковке словаря через ```**``` ключи должны соответствовать именам параметров функции

## 5. Типичные применения
- Функции с переменным количеством аргументов

- Декораторы

- Передача аргументов в другие функции

- Обработка конфигураций
